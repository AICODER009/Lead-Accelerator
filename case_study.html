Project Context

Proprietary IP Disclosure

This repository houses a **lightweight, fully functional developer demo** of the platform. The complete enterprise production build contains proprietary corporate integrations, customer CRM webhooks, and private VPC security networks. By packaging the stateful LangGraph engine and local A2A FastAPI microservices in this demo, I demonstrate the system's operational mechanics without exposing restricted business code.

Outbound outbound sales development (SDR) campaigns are notoriously resource-intensive. Generic templates fall flat, while manual research takes hours per account. I engineered this system to automate the entire lifecycle—scaling search, grounding value hooks, and stress-testing outbound copy before it is ever sent to a prospect.

Decoupled Serverless Architecture

In production, the application is architected as a high-concurrency, serverless multi-container stack deployed using **Google Cloud Run** and Vercel.

By isolating our core orchestrator from individual agent tasks, I created a decoupled architecture where heavy computations (like web research and adversarial simulations) operate as separate microservices. The services scale to zero when idle, keeping cloud resource footprints and overall compute costs minimal.

The Stateful Outbound Lifecycle

The Campaign graph transitions statefully through 5 logical nodes, utilizing persistent checkpoint transactions to ensure the workflow is completely recoverable.

01

Lead Researcher

Ingests campaign parameters and automatically compiles prospect ICP targets and descriptions.

02

Human Approval Gate

Pauses execution, serializes system state, and waits for a manager's override on the dashboard.

03

SDR Personalizer

Calls CrewAI tools to conduct semantic web searches and draft deeply grounded outbound pitches.

04

Objection Simulator

Runs dual-LLM adversarial loops, playing the role of a skeptical prospect to stress-test email copy.

05

CRM Coach

Grades response resiliency. Automatically loops back to personalization if score is subpar.

Enterprise Engineering Hurdles Resolved

Developing an enterprise-grade agent workflow required solving deep operational and network constraints:

Asynchronous Task Scaling

Timeouts

Running intensive multi-agent web scraping and research loops synchronously over HTTP resulted in server timeouts during heavy traffic periods.

Resolution

Decoupled execution by implementing a **GCP Pub/Sub task queue**. The personalization node posts jobs, transitions to a suspended state, and frees the gateway. A background worker container processes the research, updating the state once completed.

Bulletproof Session State

Fault Tolerance

With serverless instances scaling up and down, campaigns faced risk of data loss mid-workflow if an active container was terminated, resulting in wasted API spend.

Resolution

Configured a highly resilient **PostgreSQL Cloud SQL checkpointer**. Because every node step transition is executed as a database transaction, an interrupted container simply reads the state history from SQL on wake and resumes immediately.

Real-Time Inline Guardrails

Quality Checks

Relying solely on raw LLM prompts poses risk of hallucinating company statistics or fabricating metrics not supported by the client’s internal case studies.

Resolution

Engineered an **automated in-line validation filter** directly into the SDR compilation node. Using **DeepEval**, the system grades generated copy for **Faithfulness** against source text, blocking and re-routing any copy scoring below $0.85$.

💡 Value For an AI-First Product Direction

This project demonstrates a thorough understanding of what it takes to shift AI prototypes into robust, production-ready enterprise systems:

  • State Graphs Over Chains: LangGraph structures complex, predictable self-correcting logic.
  • Decoupled Sockets: Isolating microservices reduces server load and keeps compute highly optimized.
  • Human Oversight Gates: Database checkpointers guarantee brand control without slowing campaigns.
